## 2. Core Dashboards Admin API

### Proposito
Permitir a usuarios SysOwner crear y gestionar dashboards globales (templates) que pueden ser copiados a cualquier tenant.

### Endpoints Implementados

| Metodo | Endpoint | Descripcion | Permiso |
|--------|----------|-------------|---------|
| `POST` | `/api/admin/core-dashboards` | Copiar dashboard de tenant a core | SysOwner |
| `DELETE` | `/api/admin/core-dashboards/{id}` | Eliminar dashboard core | SysOwner |
| `GET` | `/api/admin/core-dashboards` | Listar dashboards core | SysOwner |
| `POST` | `/api/admin/core-dashboard-folders` | Crear folder en core | SysOwner |
| `DELETE` | `/api/admin/core-dashboard-folders/{id}` | Eliminar folder core | SysOwner |
| `GET` | `/api/admin/core-dashboard-folders` | Listar folders core | SysOwner |

### Colecciones Firestore
- `core_dashboards` - Dashboards globales/templates (sin tenant_id)
- `core_dashboard_folders` - Folders para organizar dashboards core

### Modelos de Datos

**CoreDashboard:**
```json
{
  "id": "auto-generated",
  "name": "Dashboard Name",
  "folderId": "optional-folder-id",
  "config": { /* dashboard config */ },
  "widgets": [ /* widget array */ ],
  "createdAt": "timestamp",
  "createdBy": "user-uid",
  "sourceInfo": {
    "tenantId": "original-tenant",
    "dashboardId": "original-dashboard-id"
  }
}
```

**CoreDashboardFolder:**
```json
{
  "id": "auto-generated",
  "name": "Folder Name",
  "parentId": "optional-parent-folder-id",
  "createdAt": "timestamp",
  "createdBy": "user-uid"
}
```

### Ejemplos de Uso (Frontend)

**Copiar Dashboard a Core:**
```javascript
const response = await fetch('/api/admin/core-dashboards', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    tenant_id: 'TENANT_XXX',
    dashboard_id: 'dashboard_id',
    folder_id: 'optional_folder_id'
  })
});
const coreDashboard = await response.json();
```

**Crear Folder:**
```javascript
const response = await fetch('/api/admin/core-dashboard-folders', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Templates Meta Ads',
    parent_id: null
  })
});
```

**Listar Core Dashboards:**
```javascript
const response = await fetch('/api/admin/core-dashboards', {
  headers: { 'Authorization': `Bearer ${token}` }
});
const { dashboards } = await response.json();
```

### Verificacion de Permisos
```python
def verify_sys_owner(user_info: dict) -> bool:
    """Check if user has SysOwner privileges"""
    return user_info.get("sys_owner", False) or user_info.get("role") == "SysOwner"
```

## 4. Problemas Resueltos

### 4.1 RLS Policies Bloqueando DELETE
- **Problema**: BigQuery Row Level Security bloqueaba operaciones DELETE
- **Solucion**: Script modificado para solo hacer APPEND (skip delete)

### 4.2 Tipo de Datos Mismatch (INT64 vs STRING)
- **Problema**: IDs en CSV como integers, BigQuery esperaba STRING
- **Solucion**: `pd.read_csv(csv_path, dtype=str)` para leer todos los IDs como strings

### 4.3 Orden de Columnas CSV vs BigQuery
- **Problema**: CSV tenia columnas en orden diferente al schema de BigQuery
- **Solucion**: Conversion a JSONL que usa nombres de campos, no posicion

### 4.4 Backend Signal 11 (SIGSEGV)
- **Problema**: Container crasheando con segmentation fault
- **Solucion**: Redeploy completo del servicio Cloud Run

## 6. Period Over Period (Documentado)

Se diseño la especificacion completa para comparaciones de periodos temporales.

**Documento**: `docs/PERIOD_OVER_PERIOD_SPEC.md`

### Tipos de Comparacion
| Tipo | Descripcion | Ejemplo |
|------|-------------|---------|
| same_point | Mismo punto periodo anterior | Dic 1-16 vs Nov 1-16 |
| full_previous | Periodo anterior completo | Dic 1-16 vs Nov 1-30 |
| same_point_yoy | Mismo punto año anterior | Dic 1-16 vs Dic 1-16 2023 |
| custom | Rango personalizado | Usuario define |

### Decision de UX
- **MVP**: Selector hibrido con fechas concretas (Opcion D)
- **Futuro**: Timeline visual arrastrable (Opcion C)

### Principio Clave
El selector debe mostrar **fechas concretas**, no solo labels como "periodo anterior".
El usuario debe ver exactamente que se esta comparando antes de seleccionar.

```
┌─────────────────────────────────────────────────┐
│  📅 Dic 1 - Dic 16, 2024  ▼  │  🔄 vs Nov 1-16 ▼  │
└─────────────────────────────────────────────────┘
```

### Arquitectura
- Frontend calcula rangos y hace 2 queries paralelas
- Backend no requiere cambios (endpoints actuales soportan filtros de fecha)
- Frontend calcula deltas y renderiza
